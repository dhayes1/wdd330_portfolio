TDD (Test-Driven Development)

TDD (Test-Driven Delopement) is the process of developing and writing tests before code is written. These test will initially fail because there is no code. The developer can then write code to pass the tests. Once the code passes the test, the developer can then optimize/refactor the code while continuing to ensure the tests continue to pass.

Workflow:

  1. Write tests (will initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features
